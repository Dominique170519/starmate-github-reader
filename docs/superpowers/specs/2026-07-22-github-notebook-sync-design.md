# GitHub 账号笔记卡片与跨端同步设计

## 背景与目标

当前 Chrome 插件把笔记存入 `chrome.storage.local`，Web App 把笔记存入页面 `localStorage`。两端数据模型和存储位置不同，因此无法同步；同时，两端都以一个大文本框为主，用户只能持续追加内容，难以分类、检索和复习。

本功能把笔记升级为独立卡片，并在用户主动开启后，通过 GitHub 身份在插件、电脑网页和手机网页之间同步。产品仍采用本地优先策略：未登录、离线或关闭同步时，笔记照常可用。

## 身份与同步边界

用户必须通过真正的 GitHub OAuth 登录，不能把手动填写的 GitHub 用户名当作身份凭证。Web App 创建登录会话；插件通过一次性连接流程打开 Web App 完成授权，换取仅供星伴读同步使用、可以撤销的插件凭证。

同步默认关闭。首次开启前说明将同步的数据类型、用途和删除方式。关闭同步后不再上传新变化，但保留本地数据；用户可以单独删除云端副本。

Vercel 部署使用持久化关系数据库保存用户、设备、笔记和版本。GitHub OAuth Client Secret、数据库连接和令牌签名密钥只存在于服务端环境变量。第一版采用 HTTPS 传输与托管数据库静态加密，不宣称端到端加密。

## 笔记数据模型

每条笔记是一个独立实体，至少包含：

- `id`：客户端生成的全局唯一标识；
- `userId`：同步开启后关联的 GitHub 用户内部标识；
- `repositoryId`、`documentId`、`sectionId` 和可选的原文锚点；
- `type`：`freeform`、`quote`、`understanding`、`question`、`term`、`mentor-answer` 或 `review`；
- 可选原文摘录、用户正文和来源 URL；
- 主题标签；
- `pinned`、`resolved`、`reviewNeeded`；
- `createdAt`、`updatedAt`、`version` 和可选 `deletedAt`。

笔记默认按文章分组。主题标签提供跨文章汇总，因此用户既可以从正在阅读的文章进入，也可以在“我的笔记”工作区按主题学习。

## 用户体验

插件和 Web 阅读器都支持：

- 新建空白笔记，不要求先选择原文；
- 把选中原文生成“摘录”卡片；
- 把术语解释、自己的理解、疑问或伴读回答生成对应类型卡片；
- 编辑正文、添加标签、置顶、标记已解决或待复习；
- 按当前文档查看笔记，并打开完整笔记库。

Web App 增加独立“我的笔记”工作区，支持按文章、标签和类型筛选，全文搜索，编辑、软删除和恢复上一版本。阅读器侧栏只显示当前文章的紧凑列表，避免挤占原文空间。

现有大文本笔记在首次升级时迁移为一张 `freeform` 类型的“历史笔记”卡片。迁移成功前不删除旧键，确保升级失败时仍可恢复。

## 同步流程

客户端每次编辑先写入本地笔记库，并加入待同步队列。后台在已登录且联网时批量发送增量变更。服务端验证用户、设备、版本和字段大小，写入当前笔记并保存上一版本。

客户端按游标拉取其他设备的增量变化。没有冲突时直接应用；两个设备都编辑同一版本时，以更新时间较新的内容作为当前版本，同时把被覆盖版本保存在历史中并提示用户可恢复。删除采用墓碑记录，避免离线设备把已删除笔记重新上传。

插件连接流程使用短时一次性代码，代码只能兑换一次并迅速过期。插件凭证只允许访问当前用户的笔记同步接口，不能代替用户调用 GitHub API。

## API 与组件边界

- 身份模块：GitHub OAuth、Web 会话、插件连接代码与凭证撤销。
- 笔记领域模块：类型校验、迁移、合并、冲突和版本规则，保持为可独立测试的纯逻辑。
- `/api/notes`：增量拉取、批量推送、删除和版本恢复。
- Web 笔记仓库：封装 IndexedDB 或本地存储、同步队列和 API，不让页面组件直接操作存储键。
- Extension 笔记仓库：封装 `chrome.storage.local`、同步队列和相同的 API 数据结构。
- UI：阅读器紧凑卡片、插件当前文档卡片和 Web 完整笔记库。

## 错误处理与安全

- 未登录或同步失败时继续本地保存，并显示“等待同步”，不阻塞阅读。
- 凭证过期时暂停同步并要求重新连接，不删除本地笔记。
- 每条正文、摘录、标签数量和批量请求都设置上限。
- 服务端所有查询按认证用户隔离；客户端提交的 `userId` 不作为授权依据。
- 日志不记录笔记正文、原文摘录、OAuth Token 或插件凭证。
- 用户撤销设备后，该设备凭证立即失效；删除云端数据后保留短期墓碑以完成多端删除传播。

## 测试与验收

- 旧插件和 Web 文本笔记都能无损迁移为卡片。
- 插件新建、编辑或删除笔记后，Web App 能拉取同一结果；反向同步同样成立。
- 未登录、关闭同步和离线状态下仍能正常记笔记。
- 同一笔记双端并发编辑不会丢失两个版本。
- 不同 GitHub 用户无法读取或修改彼此笔记。
- 插件连接代码不可重复兑换，撤销设备后旧凭证失效。
- 按文章、标签、类型和全文搜索均返回正确结果。
- 手机页面可以浏览、创建和编辑笔记卡片。

## 发布依赖

代码合入不等于云同步正式可用。正式发布前需要创建 GitHub OAuth App、持久化数据库和 Vercel 环境变量，并执行数据库迁移。缺少这些配置时，产品必须明确显示“仅保存在本设备”，不能伪装成已同步。
