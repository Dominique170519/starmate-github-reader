# 星伴读 Chrome 伴读侧栏（实验版）

星伴读帮助技术小白在原网页中轻松阅读 GitHub 项目和技术文档。基础能力完全在浏览器本地运行，不需要模型 API，也不会把笔记、选中原文或阅读历史上传到外部服务。

## 支持哪些页面

- `github.com` 仓库、README 和 Markdown 文件：安装后自动启用。
- Docsify / GitHub Pages 文档：先打开目标页面，再点击浏览器工具栏中的“星伴读”图标，只授权当前 GitHub Pages 网站。以后再次访问该网站会自动启用。
- 其他技术文档网站暂未适配；插件不会申请所有网站的访问权限。

## 阅读时可以做什么

- 页面顶部显示细进度条、本章进度和预计剩余时间。
- 侧栏记录当前章节、专注阅读时长、完成状态和上次阅读位置。
- 从真实标题生成文章地图，点击章节直接回到原文。
- 在当前页面搜索原文，并把选中的原文加入设备本地笔记。
- 每章最多轻提示 6 个已知专业名词；点击后查看“简单解释、生活类比、在这段话里的作用”。
- 把术语标记为“我懂了”或“需要回看”。插件只解释本地词典中有规则的词，不会临时编造解释。
- 第一次阅读单篇文档就生成“项目—文档—章节—概念”图谱；继续阅读其他文档后，通过共同概念形成跨文档关系。每条关系保留原文证据链接。
- 保存当前文档的版本快照。重新打开后显示作者新增、修改、删除的章节以及对应 GitHub 提交。
- 一键把当前仓库加入星伴读 Web App，生成更完整的大纲、无痛入门和学习路径。

## 作者更新怎样检查

插件每天最多后台检查 20 篇最近阅读的文档，只比较 GitHub 版本号；完整章节变化会在下次打开文档时计算。GitHub 限流或网络失败时会保留旧记录，稍后再试，不会把“检查失败”误报成作者更新。

## 隐私与本地数据

以下内容默认保存在当前浏览器的 `chrome.storage.local`：

- 阅读进度、专注阅读时长和完成状态
- “我懂了”和“需要回看”的术语
- 文档笔记和选中的原文
- 文档快照、作者更新记录和本地图谱

只有你主动点击“加入星伴读知识库”时，插件才会把公开 GitHub 仓库链接带到 Web App；不会自动同步私人笔记。

## 本地安装

1. 解压 `starmate-chrome-extension.zip`。
2. 在 Chrome 地址栏打开 `chrome://extensions`。
3. 开启右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”，选择解压后的文件夹。
5. 打开任意 GitHub 仓库，右下角出现“✦ 伴读”即安装成功。

要在 GitHub Pages 文档中使用，请把星伴读固定到浏览器工具栏，打开目标文档后点击一次图标并同意当前网站授权。

## 重置数据

在 `chrome://extensions` 中找到星伴读，进入“详细信息”后清除扩展数据；或移除扩展并重新安装。清除后，本地笔记、阅读历史、更新快照和图谱都无法恢复。

这是可安装的实验版 ZIP，尚未发布到 Chrome Web Store。
